// PROJECT CASE STUDY

Brickify.

A web application built for the hospital of Geel in collaboration with Dotnet Lab — digitising a paper-based Lego lending system for children. Built in a team of 3 using .NET, Angular and Azure as part of the Software Engineering course.

// 01 · OVERVIEW

Overview.

01 / PROBLEM

The hospital used a physical booklet to show children which Lego sets were available to borrow. Requests were handled manually — there was no digital overview, no way to manage the catalogue online, and no self-service for patients.

02 / ASSIGNMENT

Build a web application for the hospital of Geel that lets children browse and request Lego sets online, and gives staff a way to manage the catalogue — replacing the paper booklet entirely.

03 / SOLUTION

A full-stack web application with a .NET REST API, SQL database and Angular frontend. Children can browse available sets and submit requests. Staff and admins can manage the catalogue and handle requests through a dedicated management interface.

// 02 · FEATURES

Features.

🧱
Lego catalogue

Browse the full catalogue of available Lego sets — replacing the physical booklet with a searchable, visual online overview.

📬
Set requests

Children can select a set and submit a request directly from the app — no paper form, no manual process.

⚙️
Catalogue management

Staff can add, edit and remove Lego sets from the catalogue through a dedicated management interface.

📋
Request management

Admins have a full overview of all incoming requests and can manage their status.

🔐
Role-based access

Separate views and permissions for children browsing sets, staff managing the catalogue and admins overseeing requests.

☁️
Azure deployment

The full application is deployed on Azure with a CI/CD pipeline — production-ready and accessible from any device.

// 03 · ARCHITECTURE

Architecture.

.NET REST API
.NET · C#

The backend is built with .NET, exposing a REST API for all catalogue and request operations. Handles business logic, authentication and data access — structured to be maintainable across a team.

Angular Frontend
Angular · TypeScript

The frontend is built with Angular, providing separate views for patients and staff. The component-based architecture kept the codebase organised across a team of 3.

SQL Database
SQL · EF Core

All catalogue data, user accounts and requests are stored in a SQL database. The schema is managed alongside the codebase and version-controlled through migrations.

Azure & CI/CD
Azure · GitHub Actions

The application is deployed on Azure with a full CI/CD pipeline via GitHub Actions. Every push to main triggers a build, test and deployment — keeping the production environment always up to date.

// 05 · CHALLENGES

Challenges.

01 / TEAM COMPOSITION CHANGES

The team started as 6, dropped to 3, merged with another team back to 6, then ended as 3 again. Each change disrupted the workflow and required redistribution of ownership.

→ Approach

Kept the codebase modular and well-documented so that onboarding and offboarding team members caused as little disruption as possible. Clear branching and PR conventions helped regardless of team size.

02 / INTERNATIONAL TEAM MEMBER

Working with an international student required communicating entirely in English. His technical level was lower than the rest of the team, which meant certain parts required more guidance and some work had to be redistributed.

→ Approach

Assigned tasks that matched his skill level and paired on more complex parts. The language barrier was manageable — clear written communication in PRs and commit messages helped keep everyone aligned.

03 / SCOPE OF SOFTWARE ENGINEERING

The Software Engineering course required not just working software but a fully considered development process — architecture decisions, documentation, testing, deployment and client communication all had to come together.

→ Approach

Treated the process as seriously as the code. Set up the CI/CD pipeline and Azure environment early, wrote documentation alongside features rather than at the end, and kept the client updated with working demos.
